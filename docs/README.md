# Project Documentation (`docs/`)

Use this folder as the central place for all project documents, reports, and evidence.

## Structure

- `01_Project_Overview/`
  - Vision, scope, objectives, executive summary.
- `02_Requirements/`
  - BRD, SRS, user stories, acceptance criteria.
- `03_Architecture/`
  - System architecture, diagrams, API contracts.
- `04_Design/`
  - UI/UX docs, wireframes, design decisions.
- `05_Development/`
  - Implementation notes, module-level technical docs.
- `06_Testing_QA/`
  - Test plans, test cases, QA reports, bug summaries.
- `07_Deployment_Ops/`
  - Deployment runbooks, environment configs, SOPs.
- `08_Reports/`
  - Final report, interim reports, status reports.
- `09_Presentations/`
  - PPT/PPTX files for demos and reviews.
- `10_Meeting_Notes/`
  - MOM notes, decisions, action items.
- `11_Screenshots/`
  - App/web screenshots grouped by feature/release.
- `12_Reference/`
  - External references, research notes, links.

## File Naming Convention

Use clear and sortable names:
- `YYYY-MM-DD_topic_version.ext`
- Example: `2026-02-27_final-report_v1.docx`

## Recommended Formats

- Documents: `.docx`, `.pdf`, `.md`
- Presentations: `.ppt`, `.pptx`, `.pdf`
- Images: `.png`, `.jpg`

## Ownership

Keep each document updated with:
- author
- last updated date
- version
